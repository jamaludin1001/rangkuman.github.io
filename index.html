GIT

  • GIT adalah salah satu version control system yang diciptakan oleh Linus Torvalds
  • GIT mencatat setiap perubahan file di code atau sourcode kita

Fundemental

    Perintah perintah dasar yang sering dilakukan untuk mengoperasikan GIT
  1. git init: yaitu untuk membuat folder local sebagai project git
  2. git add: yaitu untuk mendaftakan file2 yang sudah dimodifikasi sebelum commit
  3. git commit: yaitu untuk mencatat perubahan yang dilakukan
  4. git status: yaitu untuk mengetahui status dari git kita
  5. git log: yaitu untuk melihat riwayat commit, Tekan q untuk keluar dari log
  6. git clone: digunakan untuk mengcopy repo dari server
  7. git remote: Semacam api untuk berinteraksi dari client dengan server
  8. git push: Untuk mengirimkan commit ke server
  9. git pull: Untuk mengambil commit dari server jika ada perubahan di server yang dibuat oleh branch lain

Branch

Cabang / copyan dari repo

  1. Git branch: melihat semua branch (local)
  2. Git branch --all: melihat semua branch (public)
  3. Git branch nama_branch: membuat branch baru
  4. Git checkout nama_branch: untuk berpindah branch
  5. Git branch -d nama_branch: untuk menghapus branch

Git merge nama_branch

  1. untuk menggabungkan commit dari 2 branch
  2. Relatif terhadap branch yang aktif saat ini
  3. Untuk melakukan merge harus dipastikan tidak ada file yang belum di commit

Conflict

  1. Kondisi yang terjadi ketika merging 2 branch yang memiliki perubahan pada file yang sama
  2. Ketika conflict, kita tidak bisa melakukan commit
  3. Untuk melakukan commit, conflict harus diselesaikan dulu
